== Bootstrap
// ─────────────────────────────────────────────────────────────────────────────

[source,sh]
----
cd tooling
uv sync          # resolves deps, creates .venv, installs sdlos-tooling (editable)
----

auto venv: No `pip install`, no `activate`.  Every subsequent command is prefixed with
`uv run` and uses the managed venv automatically or manual: `pip3 install`  or  `poetry` the requirements and activate venv.

[source,sh]
----
uv run sdlos version      # sdlos-tooling 0.1.0
uv run sdlos templates    # lists: camera  minimal  shader
uv run pytest             # all tests green;
----

== Create
// ─────────────────────────────────────────────────────────────────────────────

[source,sh]
----
uv run sdlos create                  # fully interactive (TTY prompts)
uv run sdlos create hello            # name given, prompts for the rest
uv run sdlos create viz --template shader --win-w 1280 --win-h 800 --no-interactive
uv run sdlos create mycam --template camera --data-dir --with-model ../assets/model.glb
uv run sdlos create myapp --template minimal --dry-run
----

Each app is a self-contained directory under `examples/apps/<name>/`: jade scene,
CSS, behaviour `.cxx`, and an auto-discovered `.cmake` — no manual `CMakeLists.txt`
edits required.  The root cmake glob picks up new `.cmake` files at *configure time*;
run `sdlos run <name> --reconfigure` after `sdlos create` on an already-configured build.

=== Options

[cols="2,1,4"]
|===
|Flag |Default |Description

|`-t / --template`      |`minimal`   |`minimal` \| `shader` \| `camera`
|`--win-w / --win-h`    |engine default |Debug window size (px)
|`--data-dir`           |off         |Create `data/shaders msl spirv img models` skeleton
|`--with-model PATH`    |—           |Copy `.glb/.gltf` into `data/models/`
|`--overwrite`          |off         |Re-scaffold; user-region code is preserved verbatim
|`--dry-run`            |off         |Print actions without writing
|`--config YAML`        |—           |Seed options from a YAML file (CLI flags win)
|`--save-config YAML`   |—           |Write resolved config for reuse
|===

=== User regions and `--overwrite`

Generated `.cc` files contain region markers:

[source,cpp]
----
// --- enter the forrest ---
// your code here
// --- back to the sea ---
----

`sdlos create <name> --overwrite` re-renders the scaffold, splices your code back
in by position, writes atomically, and keeps a `.bak` sibling.  Safe after any
template upgrade.



=== Templates

[cols="1,4"]
|===
|`minimal`  |Empty `jade_app_init` stub with state struct and commented examples.
|`shader`   |Three-column layout: preset chips \| shader canvas \| param steppers.
|`camera`   |Three-column layout: filter chips \| live video canvas \| draggable numbers.
|===

== Build & Watch
// ─────────────────────────────────────────────────────────────────────────────

`sdlos run` handles configure → build → launch → (optional) watch.
Only the named app target is rebuilt; shared engine and vendored deps are never touched.
cmake configure runs automatically when the build dir is absent or has no `CMakeCache.txt`.

[source,sh]
----
uv run sdlos run calc                          # build + launch
uv run sdlos run calc --watch                  # rebuild + relaunch on file change
uv run sdlos run calc --clean                  # clean-first build (no dep re-fetch)
uv run sdlos run viz --reconfigure             # force cmake configure (after sdlos create)
uv run sdlos run viz --reconfigure --preset macos-debug --watch
uv run sdlos run viz --no-build                # relaunch last binary
uv run sdlos run viz --build-dir ../build-asan --watch
----

[cols="2,1,4"]
|===
|Flag |Default |Description

|`-b / --build-dir`   |`<root>/build`  |cmake binary directory
|`--clean`            |off             |`cmake --build --clean-first` (no dep re-fetch)
|`-w / --watch`       |off             |Rebuild + relaunch on `examples/apps/<name>/` changes
|`-j / --jobs N`      |cmake default   |Parallel build jobs
|`--no-build`         |off             |Skip cmake; launch the last compiled binary
|`-r / --reconfigure` |off             |Force cmake configure before building
|`--preset`           |none            |cmake preset (e.g. `macos-debug`, `macos-release`)
|`-q / --quiet`       |off             |Suppress sdlos status lines
|===

== Mesh Tools in sdlos
// ─────────────────────────────────────────────────────────────────────────────

`sdlos mesh` offers a set of tools for working with mesh data.

[source,sh]
----
➜  userspace git:(main) ✗ uv run --project tooling sdlos mesh cache clear
  Cleared 6 cached file(s).
----

[source,sh]
----
uv run --project tooling sdlos mesh generate --preset mainz --no-cache
----

The **bbox is the only knob that controls building count**.
Everything else (`--lod`, `--face-count`, `--building-height`) affects appearance.

---

## 1. Expand the bounding box (biggest impact)

The presets are intentionally conservative (~0.5–2 km²). Each degree of lat/lon is ~111 km, so expanding by `0.010°` on each side roughly **4×** the covered area:

For any preset you can just override the bbox directly — the `--preset` and `--bbox` flags are mutually exclusive, so copy the preset bbox and widen it manually.

## denser OSM coverage

Some cities have far more buildings tagged per km² than others. Best ones for density

**TL;DR**: some areas generated have very few OSM-tagged buildings: a bigger `--bbox` covering a wider block of the same city.


[source,bash]
----
uv run --project tooling sdlos mesh generate \
  --preset mexico-popo --volcano --volcano-height 5426 --volcano-radius 8000 --app clima --name mexico-popo
----

== Release

=== Bundle

[source,python]
----
from sdlos.features import Bundler
bundler = Bundler(format="zip", include_data=True)
bundler.apply({"cfg": cfg, "app_dir": app_dir, "build_dir": build_dir, "root_dir": root_dir})
# context["bundle_path"] → build/dist/<name>-<date>.zip
----

Formats: `zip` · `tar.gz` · `dir`.
Binary discovery: `<build>/<name>` → macOS `.app` bundle → `bin/` → `Debug/Release/` →
recursive scan.

=== Auto-updater

[source,yaml]
----
# config/app.yaml
updater_url: https://updates.example.com/my_app
updater_channel: stable
updater_binary: tools/updater/bin/updater   # optional
----

[source,python]
----
from sdlos.features.updater import Updater, UpdaterContext
updater = Updater.from_cfg(cfg)
if updater.is_active():
    ctx = UpdaterContext.from_cfg(cfg, app_dir)
    updater.apply(ctx)   # writes data/updater.json; ctx.cmake_lines ready to append
----

== Reference
// ─────────────────────────────────────────────────────────────────────────────

=== `sdlos.core.cmake`

`find_project_root(start)` — walk upward until `CMakeLists.txt` contains
`project(sdlos`; falls back to the package ancestor chain. +
`cmake_snippet(name, win_w, win_h, has_data)` — render `sdlos_jade_app()` block. +
`write_app_cmake(app_dir, snippet, cfg)` — write `<name>.cmake` into the app dir.

=== `sdlos.core.fs`

`write_atomic(path, data)` — temp file + `os.replace()`, parents auto-created. +
`backup_file(path)` — `<name>.bak` sibling; returns `None` if source absent. +
`extract_user_regions(text)` / `splice_user_regions(generated, existing)` — region splice.
